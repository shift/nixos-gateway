# Policy-Based Routing Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement advanced policy-based routing with support for time-based rules, native nftables integration, and a comprehensive multi-router test suite.

**Architecture:** Extend the existing `policy-routing` module to support complex matching criteria (like time) by generating hybrid `nftables` marking rules and `ip rule` lookup commands. Verify using a 4-node NixOS test topology (Gateway, Client, ISP1, ISP2).

**Tech Stack:** NixOS Modules, iproute2 (`ip rule`, `ip route`), nftables (`meta mark`, `meta time`), Python (NixOS Test Driver).

### Task 1: Extend Policy Library for Complex Rules

**Files:**
- Modify: `lib/policy-routing.nix`

**Step 1: Create failing test (Conceptual - verify logic)**
We need to verify that we can detect "complex" rules (those needing nftables).

**Step 2: Update `lib/policy-routing.nix`**
Add logic to:
1.  Identify "complex" matches (currently: `time`).
2.  Assign unique `fwmark` IDs to complex rules (starting at `0x1000` + index).
3.  Generate `nftables` rules for complex matches.
4.  Generate `ip rule` commands that use `fwmark` for complex matches instead of direct selectors.

**Step 3: Commit**
```bash
git add lib/policy-routing.nix
git commit -m "feat(lib): add complex rule generation logic for policy routing"
```

### Task 2: Update Policy Module Interface

**Files:**
- Modify: `modules/policy-routing.nix`

**Step 1: Add Time Option**
Add the `time` submodule to the `match` options:
```nix
time = {
  start = lib.mkOption { type = types.str; description = "Start time (UTC), e.g. 08:00"; };
  end = lib.mkOption { type = types.str; description = "End time (UTC), e.g. 17:00"; };
  days = lib.mkOption { type = types.listOf types.str; default = ["Mon" "Tue" "Wed" "Thu" "Fri"]; };
};
```

**Step 2: Integrate Nftables Generation**
Inject the generated nftables rules into `networking.nftables.tables`.
- Create a new chain `chain prerouting { type filter hook prerouting priority mangle; ... }`.
- Add the rules generated by `lib.policy-routing.nix`.

**Step 3: Commit**
```bash
git add modules/policy-routing.nix
git commit -m "feat(module): add time-based policy routing options and nftables integration"
```

### Task 3: Implement Multi-Router Test Suite

**Files:**
- Modify: `tests/policy-routing-test.nix`

**Step 1: Define Topology**
Create a 4-node network:
- `gateway`: The DUT.
- `client`: Connected to gateway LAN.
- `isp1`: Upstream 1.
- `isp2`: Upstream 2.

**Step 2: Define Test Script**
1.  **Connectivity Check:** Verify `client` can ping `isp1` and `isp2` (default routing).
2.  **PBR Verification:**
    - Apply PBR: HTTP -> ISP1, HTTPS -> ISP2.
    - Run `tcpdump` on ISPs.
    - Generate traffic from `client`.
    - Assert traffic arrives on correct ISP interface.

**Step 3: Run Test**
`nix build .#checks.x86_64-linux.policy-routing-test`

**Step 4: Commit**
```bash
git add tests/policy-routing-test.nix
git commit -m "test: add multi-router topology for policy routing verification"
```
